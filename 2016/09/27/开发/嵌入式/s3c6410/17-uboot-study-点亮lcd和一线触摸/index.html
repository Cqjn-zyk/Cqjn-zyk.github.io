S3C6410的显示控制器

显示控制器有一个用于转换图像数据的逻辑，这个逻辑是指从本地总线的后处理器或系统内存内的视频缓冲区到外部LCD驱动器接口的传输图像数据的逻辑。

LCD驱动支持四种接口：
  • 传统的RGB接口
  • I80接口
  • NTSC/PAL标准TV编码器接口
  • IT-R BT.601 接口
    这里使用的lcd是RGB接口的，因此后面主要介绍RGB接口。
显示控制器由VSFR、VDMA、VPRCS/VTIME 和视频时钟产生器组成:
  • VSFR 包括可编程寄存器设置和2个256*25的调色板存储器，用于配置显示控制器
  • VDMA 专用于显示DMA，VDMA可以将帧存储器内的视频数据转换到VPRCS。通过使用特殊的DMA-VDMA，可以不使用CPU，直接将视频数据显示在屏幕上
  • VPRCS 接受VDMA发出的视频数据，将其转换到适合的数据格式，如8 位像素或16 位像素后，将视频数据发送到显示设备上
  • VTIME 包括可编程逻辑，以支持在不同LCD 驱动上发现的接口时序和速率的各种环境。VIIME 模块产生RGB_VSYNC, RGB_HSYNC, RGB_VCLK, RBG_VDEN, SYS_CS,SYS_CS0 等等。
RGB 接口说明
  • TIME时序： 由VTIME产生控制信号，如 RGBZ-VSYNC,RGB_HSYNC,RGB_VDEN和RGB_VCLK信号，这些时序控制信号与VSFR寄存器内的VIDTCON0/1/2寄存器的配置有很大的关系，根据VSFR中的配置，VTIME可以产生lcd所需要的控制信号。
    • VSVNC 垂直同步信号,帧信号
    • HSYNC 行同步信号
    • VCLK 时钟信号
    • VDEN 使能信号
    • CS 片选信号
    • CS0 片选信号
      VCLK信号的速率可以由VIDCON0 寄存器内的CLKVAL 域来控制，VCLK（hz）=HCLK/(CLKVAL+1) CLKVAL >=1
  • IO接口： RGB 接口管脚描述

S70 7寸LCD 介绍

所用的S70 LCD 屏幕为友善之臂所做的， 用了所谓的一线触摸驱动，先来看下硬件原理图：

说明：上图所示，左边为S70 LCD板连接到开发板上的排线接口，主要引脚为LCD 的RGB接口相关的引脚和电源引脚以及复位引脚，除此之外，还多了两个引脚，TOUT1 和EINT10 引脚，这两个引脚就是友善之臂用来控制LCD 电源和背光还有得到触摸数据的。右图为lcd 所用的驱动芯片 的接口，其中大部分IO口都连接到了排线接口，还有一部分接到了电源引脚，需要注意的是：VLED+ ，VLED- VGH，VGL 引脚分别为LCD的背光灯电源和高压电源，这几个引脚在以下原理图中：

如上图：有两个升压芯片MP1541DJ 和EUP2584， 前者将5V升压到40V左右，为LCD提供高压电源，后者为LCD背光提供电源。控制这两个芯片开关的引脚有两个，一个为BL_EN引脚，此引脚同时控制两个引脚的使能，此外控制LCD背光的还有一个引脚为LEDDM，它的频率可以控制背光LED的亮度。下面来看这两个引脚连接到了哪里：

如上图：BL_EN引脚和LDDM引脚连接到了一个STC单片机上，受它控制，所以重点就是这个单片机了，它完成了控制背光和获得触摸数据的功能，它的一个SPI接口连接到了触摸屏驱动IC上来获取数据，因此，重点就是我们如何来控制这个单片机了,重点就是Tout1 和EINT10这两个引脚，它们被引出接到了S70 与开发板的排线接口，最后被连接到了处理器的两个引脚上。如下图：

上面有三个lcd 接口，因为开发板兼容了几个不同大小的屏，控制单片机的两个引脚被接到了GPE0和PWM1（GPF15）接口。其中Tout 对应GPE0，EINT10对应PWM1（GPF15），为啥用两根引脚呢，推测是当GPE0 使能时，单片机接受GPF15的信号控制，相当于使用GPE0 IO口作为一个开关，而GPF15控制单片机的协议官方并没有给出，所以我们只能从一线触摸代码中了解一些。

显示控制器寄存器描述

简单的看下用到的一些寄存器，还有很多功能性的寄存器，这里只是点亮lcd ，用不着，就不多说了，具体的等使用的时候再来查看芯片手册就行了。

  • MIFPCON 设置LCD by pass，此处设置为Normal mode
  • VIDCON0 设置接口类型，数据类型等等

  • VIDCON1 设置各种控制信号的极性
  • VIDTCON0 时序控制寄存器0 根据LCD驱动芯片datasheet中时序参数设置
  • VIDTCON1 同上
  • VIDTCON2 写入水平和垂直尺寸
  • WINCON0 支持5个窗口，我们简单使用窗口0就可以了

  • VIDOSD0A 窗口位置设置A
  • VIDOSD0B 窗口位置设置B
  • VIDOSD0C 窗口位置设置C
  • Window buffer addr start
  • Window buffer addr end
